我们一般工程开发都是用vue-cli创建项目，然后基于单vue文件的模式去进行组件开发。这种模式下，在npm run build的过程中，框架会将.vue文件里的代码编译成render函数在运行时使用。因为已经编译过了，所在我们在用的时候只要导入Vue runtime即可使用。

Vue本身它还导出了runtime + compiler的版本，这样子可以在写的时候，在html文件里通过script标签将编译好的Vue的库文件加载，然后将本html文件的代码在运行时进行解析和构建。比如下面的例子：

<div id="app">
    <input type="text" v-model="word">
    <p>{{word}}</p>
    <button v-on:click="sayHi">change model</button>
</div>

<script src="./xx/xx/xx/vue.js"></script>

<script>
    var vm = new Vue({
        el: '#app',
        data: {
            word: 'Hello World!'
        },
        methods: {
            sayHi: function() {
                this.word = 'Hi, everybody!';
            }
        }
    });
</script>

上面的代码里，Vue会查找el指定的元素，然后获取它的html string，进而解析编译成Vue程序。这就是为啥源码里有去判断el是否带#，并且拿outerHtml的地方。
